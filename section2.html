Section 2: The Date Level Folder

Scoreboards

We will now look at what I've been calling the Date Level Folder. From here you can access the daily scoreboards in .json and .xml format. There are also a few different types of scoreboards at this level:

  • master_scoreboard.(ext)
  • miniscoreboard.(ext)
  • scoreboard.(ext)
  • scoreboard_deviceName.xml
See the July 30th, 2014 folder for a complete listing.

Individual Game Folders (GID)

Once you get to this level (on a day there is a game) you can access subfolders for each individual game. These subfolders are pre-fixed with "gid", so I will refer to these as the GID folders. The syntax for the subfolders are potentially the trickiest items to figure out how to properly run a routine to obtain data from them. That syntax is as follows: 

gid_YYYY_MM_DD_XXXmlb_YYYmlb_1/

Where XXX and YYY are the static 3-letter codes for each team.
Ex: Seattle = sea.

In the case of the MLB At Bat app, or a similar item you may want to build, you would have each Scoreboard item act as a link. Clicking the link would be able to build a URL that is constructed of each team code. That would then allow a request to the GID folder for that game.

There is a lot of interesting information that can be obtained here. One file of great note would be the inning_X.json/xml files located within the inning/ directory. This contains an account of every pitch and play broken down by inning. You will see that file shows the result of each play and even the physics of each pitch.

Again, accessing this data is based on continuing to builg the URL to get down to each .json or.xml file.
