Section 1: The API Structure

As discussed in the Introduction, the base URL is:

http://gd2.mlb.com/components/game/mlb/

The basic structure begins with the base URL and then trees down first by year, then month then date. The format is similar to the following:

  • year_YYYY/
    • month_MM/
      • day_DD/

    Building the URL out from that would take us to the Date Level Folder, as mentioned in the Introduction. 

                  http://gd2.mlb.com/components/game/mlb/year_YYYY/month_MM/day_DD/

    At the year level folder you can continue on to folders for each month. But this also contains batters/ and pitchers/ folders. Within those folders are .xml files for each batter and pitcher that played that year. These files can only be accessed if you have the batter's or pitcher's ID number. That ID number is assigned by MLB.

    Accessing these files would require a .csv (for example) file that can tie batters to their ID numbers. There are a number of developers on Github whcih have compiled such lists, but I'm opting to not 'borrow' that data. So this How-To will not spend a lot of time on those files. I wanted to point them out as they are accessible and would contain relevant data depending on what you would like to do.
